03-Pure JavaScript vs React Virtual DOM | الفرق بين جافاسكربت و DOM الافتراضي في React

مقدمة

السلام عليكم، اللهم صلي وسلم وإذن وبرك على سيدنا محمد وعلى آله وأصحابه المعينين طيب، بإذن الله، النهاردة هنكمل رحلة معرفة ال-DOM واستكشاف ايه هو اللي بيحصل في البروزر وانا بستخدم ال-Vanilla JavaScript وانا بستخدم ال-React.

احنا كنا تناقشنا المرة اللي فاتت عن المراحل اللي بيتم من خلالها عمل رندر للوبسيط. رندر يعني ايه؟ يعني ان هو يظهر لك ببساطة شديدة الوبسايت بتاعك في البروزر!

شرح العملية

المراحل التي تحدث عند بناء الموقع تبدأ بتحليل ملفات HTML و CSS ثم تحويلها إلى شجرة DOM وCSSOM. يتم دمجها في النهاية ضمن Render Tree من أجل تحديد التصميم النهائي قبل رسم الشاشة.

أما عند استخدام Vanilla JavaScript، يتم التعديل على ال-DOM مباشرة، الأمر الذي يتطلب إعادة بناء كاملة لكل مرة يحدث فيها تعديل، مما يؤدي إلى بطء في الأداء.

مقارنة مع React Virtual DOM

React يقوم بإنشاء Virtual DOM وهي نسخة خفيفة من DOM الحقيقية، تُخزن في الذاكرة بحيث يتم تنفيذ التعديلات عليها أولاً قبل عكسها على الـReal DOM. ذلك يقلل من عمليات إعادة بناء الأشجار قدر الإمكان.

باستخدام الخوارزمية Diff Algorithm، يقوم React بمقارنة النسخة القديمة من Virtual DOM بالنسخة الجديدة ويحدث فقط الاختلافات بشكل مباشر.
